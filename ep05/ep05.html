In this project we are going to explore out knowledge in neural networks modeling using the scikit learn framework. The previews tasks of this discipline provided application of fundamental topics in machine learning, such as the perceptron and linear regression.

This time we are going to build a multi-layer perceptron classifier (MLP) for the MNIST dataset, hence we are going to predict labels for handwritten digits. We are going to develop this project in 3 parts:

  • Data preparation: the first part is to extract the data from the dateset and build a reduced (used in cross validation) and a complete (used in the training) set, both being reshaped and normalized to fit in the models correctly.
  • Model selection: we are going to use the 5-fold stratified cross-validation method to evaluete the best model/architeture for our network, looking for the accuracy to make our decision toward the best fitting model.
  • Selected model evaluation: Finally we are going to train the choosen model using the complete sets. During this process we will be able to take some metrics such as loss error during the training and the confusion matrix after the training.

1 Data preparation¶

The next code cells deal with the preparation of the dataset used. This part is essencial to the correct training of the network since the MLP is vary sensible to the input. All the preparation is done in this cells and nothing is left to be made later in the code.

1.1 MNIST dataset¶

This is a well known dataset, commonly used as a first example of image classification tasks. It consists of handwritten digits, divided into $60000$ training images and $10000$ test images. All images are gray-scale (one channel with pixel intensities varying from 0 to 255) and have size $784$ (the image from sklearn is already flatterned). There are 10 classes, corresponding to digits 0 to 9.

1.3 Separating reduced training set¶

Here we are going to create a reduced traning set, consisting of $n\_samples = 2000$ examples of each class. The focus of the reduced set is to decrease the amount of time and processing needed in order to make the cross validation. After this we will have:

  • x_train and y_train as the reduced sets for cross validation
  • x_test and y_test as the original test sets from the dataset
  • x_train_complete and y_train_complete as the complete sets to be used in the final training
In [4]:
from sklearn.utils import shuffle
import numpy as np

# number of samples from each class to be inserted in the reduced set
n_sample = 2000

# getting reduced test set with same amount of samples from each class
x_train_reduced = y_train_reduced = None
for c in range(10):
    itens = [i for i in range(len(x_train_features)) if int(y_train_all[i]) == c]
    x_tmp, y_tmp = shuffle(x_train_features[itens], y_train_all[itens])

    if x_train_reduced is None:
        x_train_reduced, y_train_reduced = x_tmp[:n_sample], y_tmp[:n_sample]
        continue
    
    x_train_reduced = np.concatenate([x_train_reduced, x_tmp[:n_sample]]) 
    y_train_reduced = np.concatenate([y_train_reduced, y_tmp[:n_sample]])

print("x_train_reduced shape: {0} --- y_train_reduced shape: {1}".format(x_train_reduced.shape, y_train_reduced.shape))

# getting final sets
x_train, y_train = shuffle(x_train_reduced, y_train_reduced)
x_test, y_test = x_test_features, y_test_all
x_train_complete, y_train_complete = shuffle(x_train_features, y_train_all)

2 Model selection¶

This is the most important part in our project. To chose a good model (in order words, a model/architeture/network that has a high probability of giving us the rigth label for an image) we must have a good and reliable way to mesure the accuracy of that model.

To do this we are going to use the model selection API providade by sklearn, this API enable us to make a 5-fold (divide the reduced train set into 5 groups/folds) stratified (each class will have the same number of representants in each fold) cross-validation (process made for each folder, taking the corresponding one to evaluete the error in the model trained by the others) in each one of the presented models.

To create the models itself we are going to use the MLPClassifier, a multi-layer perceptron classifier provided in sklearn that enable us to specify a big variety of hyperparameters such as the hidden layers layout, activation function, solver, number of iterations and much more.

In [11]:

2.1 Models¶

Here we are going to create our prediction models. Our models are going to be specified as lines in the list model_param, this way the process of creating new instances of a MLPClassifier will be very easy.

The sklearn easily deal with classification problems, hence in all our models the 10-sized output layer is as softmax output funtion. Also, at the current moment, it is only able to minimize the cross-entropy loss function to fit the data, in other words, to learn.

The model 0 (first in the list) is Based on Franois Chollet model presented here. The network has 512 nodes in the first layer and 10 in the second one, the batch size is of 128 samples.

Each one of the following models vary only in the number of layers, the number of nodes in each layer and the activation function.

Also, for all the models we:

  • Used the $adam$ (stochastic gradient-based optimizer) as solver, because all the other solvers perfomed badly in off-screen tests
  • Used $batch\_size=128$
  • Fixed $max\_iter=10$ for a consistent analisys of each one

2.2 5-fold cross validation¶

Now we finally are going to train some neural networks. Using the StratifiedKFold method from sklearn we divided the reduced training ser into 5 folds and though a loop each fold was used as test set while the rest were used to train the model.

That train of a model is made by the mothod fit() and after that we can measure the error between the prediction and the actual labels.

In the output console is possible to check the accuracy of each model in each fold iteration for both training and testing sets. The final accuracy is a result of the mean of the 5 observed accuracies and we are looking toward this final result to make our decision of the best architeture.

In [8]:
from collections import Counter

# to remove annoying warnings
from warnings import simplefilter
from sklearn.exceptions import ConvergenceWarning
simplefilter("ignore", category=ConvergenceWarning)

# stratified 5-fold used for cross validation
skf = StratifiedKFold(n_splits=5)
folder_number = 1
accuracy_history = [[] for m in range(number_models)]

# train and test are lists of indexes
for train, test in skf.split(x_train, y_train):
    # train and test sets for this fold
    x_train_f, y_train_f = x_train[train], y_train[train]
    x_test_f, y_test_f = x_train[test], y_train[test]

    print("---------------------------------------------------------")
    print("Class distribution in folder {0}:".format(folder_number))
    amt_train = [x[1] for x in sorted(zip(list(Counter(y_train_f).keys()), Counter(y_train_f).values()))]
    amt_test = [x[1] for x in sorted(zip(list(Counter(y_test_f).keys()), Counter(y_test_f).values()))]
    print(pd.DataFrame({'train set':amt_train, 'test_set':amt_test}, range(10)).T)
    print("")

    for m in range(number_models):
        # new model to be trained
        model = MLPClassifier(hidden_layer_sizes=model_param[m][0], activation=model_param[m][1], \
                              solver='adam', batch_size=128, max_iter=10)

        model.fit(x_train_f, y_train_f)
        train_acc = model.score(x_train_f, y_train_f)
        test_acc = model.score(x_test_f, y_test_f)
        accuracy_history[m].append([train_acc, test_acc])
        print("Model {0}: {1:.2f}% accuracy on train set and {2:.2f}% accuracy on test set".format(\
            m, train_acc*100, test_acc*100))

    folder_number += 1

print("---------------------------------------------------------")
print("\nFinal accuracy of the models:")
for m in range(number_models):
    print("Model {0}: {1:.2f}% of accuracy in train sets and {2:.2f}% of accuracy in test sets".format(\
        m, np.mean(accuracy_history[m], axis=0)[0]*100, np.mean(accuracy_history[m], axis=0)[1]*100))

3.2 Training curve¶

From the difinition in the sklearn website we have "A learning curve shows the validation and training score of an estimator for varying numbers of training samples. It is a tool to find out how much we benefit from adding more training data and whether the estimator suffers more from a variance error or a bias error.".

We are going to use the learning_curve() method to help us doing this job, wich is based on 5-fold stratified cross validation of the training set. To generate the following plots I took inspiration in this tutorial from Chris Albon.

In [22]:
from sklearn.model_selection import learning_curve

train_sizes, train_scores, test_scores = learning_curve(MLPClassifier(hidden_layer_sizes=(1024, 1024, 10), 
                                                        activation='relu', solver='adam', \
                                                        batch_size=128, max_iter=10), \
                                                        x_train, y_train, scoring='accuracy', \
                                                        cv=5)
# means and standard deviations of training set scores
train_mean, train_std = np.mean(train_scores, axis=1), np.std(train_scores, axis=1)
# means and standard deviations of test set scores
test_mean, test_std = np.mean(test_scores, axis=1), np.std(test_scores, axis=1)
plt.plot(train_sizes, train_mean, '--', color="#4081c7",  label="Training score")
plt.plot(train_sizes, test_mean, color="#57bd61", label="Cross-validation score")
plt.fill_between(train_sizes, train_mean-train_std, train_mean+train_std, color="#c7e2ff")
plt.fill_between(train_sizes, test_mean-test_std, test_mean+test_std, color="#cfffd3")
plt.title("Learning curve")
plt.xlabel("Training set size"), plt.ylabel("Accuracy score")
plt.legend(loc="best")
plt.tight_layout()
plt.show()

4 Support vector machines¶

The SVM is a very powerfullway to classify data, we can call it a improved perceptron. Sklearn has a lot of content about SVM and has a very powerfull library for it. If we have $n\_classes$ number of diferent classes in our data the classifier supports the one-versus-one method for classification, this is, it will produce ${n\_classes \choose 2}$ diferent SVMs, one for each pair of classes. But as our dataset is a little big, we are going with the one-versus-rest approach, resulting in only $n\_classes$ SVMs.

Particularly we are using the linear support vector classification class of sklearn. Here again we are going to print the accuracy of this model together with the confusion matrix. We will use the reduced train set for training
